How Assessment Connects

The assessment generator writes actual quiz/exam questions using content extracted directly from your course syllabus.

  1. Syllabus — AI extracted the weekly plan (topics, CILOs, resources) from your syllabus document.
  2. TOS — defines how many items each topic gets and the Bloom's spread (Remember → Create).
  3. Assessment (this page) — pulls syllabus content for the weeks you select and writes questions that follow the chosen Bloom's level and difficulty.
How to Generate
  1. Select weeks. The AI will use the extracted syllabus data for these weeks as source material.
  2. Pick question types. Tick the types you want (Multiple Choice, True/False, Identification, Essay, etc.) and set the item count per type.
  3. Set Bloom's level + difficulty. Match these to your TOS distribution if you've already built one for the same topics.
  4. Click Generate. Each week takes 10–20 seconds. The questions are saved to a Question Set you can preview, edit, or export.
Requirements
  • Extracted Syllabus with weekly plan data. {% if not has_syllabus %} Upload one → {% endif %}
  • At least one question type selected with an item count.
  • At least one week selected.
  • A Bloom's level (or "Mixed") and a difficulty.
  • A configured AI provider key (Groq, OpenAI, Anthropic, or Google).
  • Recommended: build a TOS first so item counts and Bloom's distribution are aligned with your exam plan.
Tips
  • Use the same topic names across presentations and TOS so the AI can match them cleanly.
  • Mix types — MCQ for Remember/Understand, Essay for Analyze/Evaluate/Create.
  • For long exams, generate by section (e.g. weeks 1–4, then 5–8) to keep AI grounding tight.
  • Edit any generated question afterwards — nothing is locked in.
